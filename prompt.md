请帮我开发一个「心理年龄测试」网站，要求**可以直接部署上线给真实用户使用**。  
后端使用 **Node.js + Express + Prisma + PostgreSQL**，前端使用原生 HTML/CSS/JS，风格温暖治愈。

## 一、核心功能

### 1. 兑换码验证与消费
- 用户点击“开始测试” → 弹出模态框输入兑换码。
- 前端调用 `/api/validate-code` 验证兑换码是否存在且 `used = false`。
- 验证通过后记录该码到前端（仅用于提交时消费），不立即标记使用。
- 用户提交全部答案时，后端 **在同一个数据库事务中** 标记兑换码为已使用，并保存测试结果。
- 若标记时发现兑换码已被使用（并发冲突），则返回错误并提示用户。

### 2. 20 道心理年龄测试题
- 题目设计覆盖 8 个维度：
  1. 情感稳定性
  2. 社交开放性
  3. 自我认知度
  4. 责任感
  5. 好奇心
  6. 适应性
  7. 乐观倾向
  8. 压力应对
- 每题 1~5 分，每个维度由若干题组成，得分线性映射到 0~10 分。
- 心理年龄通过总分映射到 18~60 岁（可自定义映射公式）。
- 8 种人格类型基于维度得分组合判定（见下方映射表）。

### 3. 结果展示与截图保存
- 结果页包含：
  - 心理年龄（大数字） + 用户填写的实际年龄（对比差值）
  - 关键词标签（3~5 个，如“成熟”“敏感”“好奇”）
  - 雷达图（Chart.js）
  - **人格类型**（名称 + 简短描述）
  - **原型角色**（如“智慧长者”“温暖治愈者”）
  - **匹配名人**（如“奥普拉·温弗瑞”“达芬奇”）
  - 深度自我分析（100~150 字个性化文字）
  - 社交与匹配建议（50 字左右）
- “保存结果图片”按钮使用 `html2canvas` 将整个结果区域截图保存。

### 4. 管理员后台
- 独立路由 `/admin`，需要 JWT 登录（用户名+密码）。
- 管理员可：
  - 生成兑换码（8 位字母数字，大写，单次最多 100 个）
  - 查看所有兑换码列表（含使用状态、使用时间、关联测试结果摘要）
  - 导出 CSV 对账

## 三、技术栈与实现要求

### 后端
- **Express** 框架，RESTful API。
- **Prisma** ORM，连接 **PostgreSQL** 数据库（生产环境推荐 Supabase 或云 PostgreSQL）。
- **数据库模型**：
  - `ExchangeCode`：id, code, used (Boolean), usedAt, createdAt, testResultId (关联)
  - `TestResult`：id, mentalAge, realAge (用户填写), dimensionScores (JSON), personalityType, archetype, matchedCelebrity, keywords (JSON), analysisText, matchText, createdAt, exchangeCodeId
- **API 路由**：
  - `POST /api/validate-code` – 验证兑换码是否有效（未使用）
  - `POST /api/submit-test` – 提交答案，消费兑换码，存储结果，返回结果数据（需在事务中完成）
  - `POST /api/admin/login` – 管理员登录，返回 JWT
  - `GET /api/admin/codes` – 获取所有兑换码（需 JWT）
  - `POST /api/admin/generate-codes` – 生成指定数量兑换码（需 JWT，限制单次最多 100 个）
  - `GET /api/admin/export` – 导出 CSV（需 JWT）
- **安全措施**：
  - 使用 `express-rate-limit` 限制 API 调用频率（例如 `/submit-test` 每个 IP 每分钟最多 3 次）。
  - CORS 限制只允许前端域名。
  - 所有输入参数使用 `express-validator` 校验。
  - 管理员密码使用 `bcrypt` 哈希存储（初始化时创建默认管理员账户，可通过环境变量配置）。

### 前端
- 原生 HTML/CSS/JS，完全独立于后端，通过 fetch 调用 API。
- 使用 **Chart.js** 绘制雷达图，**html2canvas** 实现截图。
- 温暖治愈风格：柔和的渐变色（奶油色、淡橘、淡粉），圆角卡片，手绘风格装饰，大圆润字体。
- 响应式设计，支持手机端。

### 部署准备
- 提供 `docker-compose.yml` 方便本地开发（包含 PostgreSQL + Node 应用）。
- 环境变量配置：数据库连接串、JWT_SECRET、ADMIN_PASSWORD_HASH。
- README 包含详细部署步骤（手动部署、Railway/Render 一键部署指南）。

## 四、额外要求

- **测试数据初始化**：启动时检查数据库中是否有题目，若无则自动插入 20 道预设题目（题目内容由你设计，覆盖 8 个维度）。
- **管理员首次登录**：若无管理员账户，则自动创建默认账户（账号 `admin`，密码从环境变量 `INIT_ADMIN_PASSWORD` 读取，若未设置则打印到控制台）。
- **数据备份**：无需程序内实现，但建议在文档中说明数据库备份方式。
